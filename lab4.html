Ahora, recordemos que Word2Vec ayuda a representar una palabra por su contexto, para ello necesitamos definir una ventana movil (sliding window) que se usa dentro del algoritmo. Esta consiste en tomar cada palabra de una frase, y luego se parea con las N palabras ms cercanas (hacia la derecha e izquierda). Por ejemplo, consideremos una frase como "every good dog does fine", con una ventana de 2. El resultado sera algo como:

(every, good) (every, dog) (good, every) (good, dog) (good, does) (dog, every) (dog, good) ...

Y as consecutivamente. La frase u oracin, es convertida en un par target, context donde el contex es una lista de tokens dentro de la ventana.

    """

Con esto en consideracin, es momento de implementar nuestro modelo Word2Vec.

Noten el embedding_size, este corresponde a la cantidad de representaciones de cada palabra, como dijimos en clase, esto sera la cantidad de funciones de activaciones con las que trabajaremos.

Adems, consideren las siguientes explicaciones

self.embed: Es una capa de embedding para convertir la entrada (el ndice del token de centro/contexto) en la codificacin one-hot, y luego recuperar los pesos correspondientes a estos ndices en la capa hidden de menor dimensin.

self.expand: Es una capa lineal para predecir la probabilidad de una palabra de centro/contexto dada la hidden layer. Deshabilitamos el bias (la intercepcin) porque cambiamos la escala de nuestras predicciones de todos modos.

logits: Este vuelve a expandir la capa hidden para hacer predicciones. Estas predicciones sin procesar deben volver a escalarse con softmax, pero omitimos este paso aqu, ya que PyTorch implementa los pasos relevantes en la Cross Entropy loss.

In[]:
class Word2Vec(nn.Module):
    def __init__(self, vocab_size, embedding_size):
        super().__init__()
        self.embed = nn.Embedding(vocab_size, embedding_size)
        self.expand = nn.Linear(embedding_size, vocab_size, bias=False)

    def forward(self, input_):
        # Pasamos el input a una representacin ms pequea
        hidden = self.embed(input_)
        # Expandemos hacia las predicciones
        logits = self.expand(hidden)
        return logits

Paso 3 - Entrenamiento (Training)

El entrenamiento en el contexto de las redes neuronales significa hacer predicciones repetidamente utilizando las observaciones en el conjunto de datos y luego ajustar los parmetros para corregir el error en las predicciones.

Debido a que no queremos que la red aprenda perfectamente la prediccin ms reciente mientras olvida todas las dems predicciones, generalmente le damos un "learning rate", que es una penalizacin en el ajuste de prdida para evitar que se ajuste solo a la observacin ms reciente. (Recuerden como funciona backpropgation)

Cuanto ms tiempo entrenemos la red, con mayor perfeccin aprender los datos de entrenamiento, pero a menudo esto conlleva el riesgo de overfitting y no poder generalizar a datos no vistos. RESPUESTAS:

  • En modelos como el Word2Vec, el overfitting ocurre cuando se encuentran relaciones (cercanas) entre palabras nicamente del corpus de entrenamiento, pero que no podran ser aplicadas a otros contextos. Este problema implicara que, al modificar el corpus o ingresar nuevos conjuntos de palabras, el modelo no sea capaz de identificar nuevas relaciones y, por lo tanto, no "entendera" nuevos contextos.
  • Consideramos que el modelo consigui interpretar satisfactoriamente las relaciones entre palabras, puesto que defini 4 conjuntos: good, bad, school y day; para los cuales asign diferentes palabras, y concluimos que, en su mayora, coinciden con el conjunto al que pertenecen. No obstante, tambin identificamos que hay palabras que no necesariamente deberan pertenecer a ese conjunto, o que simplemente no aplican directamente a ninguno de los grupos identificados, por lo que creemos que el modelo s podra ser mejorado, esto mediante el ajuste del corpus de entrenamiento (aadiendo ms palabras para un mayor contexto), implementando diferentes dimensionalidades para los vectores, o simplemente ajustando hyperparmetros como el learning rate. Tambin podra ser til realizar una limpieza ms detallada del conjunto de datos, ya que como se mencion al inicio del laboratorio, no se profundiz en este tema y esto podra conllevar a un menor rendimiento del modelo.
  • A grandes rasgos, la diferencia entre Word2Vec y BERT (Bidirectional Encoder, Representations from Transformers) recae en que, en el caso de Word2Vec, para cada palabra se generar el mismo vector, independientemente del contexto o situacin en la que se est utilizando. Mientras que en BERT, el vector para una misma palabra puede variar dependiendo de su contexto. De ah que BERT sea un enfoque bidireccional, puesto que no toma en cuenta solo el contexto actual de la palabra en una frase sino tambin el previo.

Introduccin

Los modelos ms comunes seq2seq son los modelos encoder-decoder, los cuales usan una RNN para encodear el input y llevarlo a un solo vector. En este laboratorio nos estaremos refiriendo a dicho vector como vector contexto. Pensemos sobre el vector contexto como un ser abstracto que representa una frase completa. Este vector es luego decodeado por una segunda RNN, que aprende a generar la frase target (output) deseada al generar palabra por palabra.

Consideren la siguiente ilustracin para representar el proceso que estaremos realizando

Crdito de imagen al autor, imagen tomada de "Sequence to Sequence Learning with Neural Networks" de Ben Trevett

Noten como la frase de input "guten morgen", se pasa a travs de una capa de embedding (cuadros amarillos) y luego entra en los encodeadores (cuadros verdes). En esta ocasin agregamos un token de "start of sequence" (<sos>) al inicio de la frase, adems de un token de "end of sequence" (<eos>) al final de la oracin. Vean como en cada paso, la entrada del encoder RNN es tanto la representacin embedding $e$ de la palabra actual $e(x_t)$, as como el estado oculto del paso anterior $h_{t-1}$, y el encoder genera un nuevo hidden state $h_t$. Entonces, podemos pensar en el hidden state como una representacin vectorial de la oracin hasta ese momento. La RNN se puede representar como una funcin de tanto $e(x_t)$ y $h_{t-1}$

$$h_t = \text{EncoderRNN}(e(x_t), h_{t-1})$$

Por favor noten que estamos usando el termino RNN de forma general en este contexto, puede ser cualquier arquitectura como LSTM o GRU.

Entonces estaremos trabajando con una secuencia como $X = \{x_1, x_2, ..., x_T\}$, donde $x_1 = <sos>$, $x_2 = guten$, y as consecutivamente. El hidden state inicial $h_0$ es usualmente iniciado con ceros o con algn parametro pre-aprendido.

Una vez la palabra final $X_T$ ha pasado en la RNN a travs de la embedding layer, usamos el hidden state final $h_T$ como vector de contexto. Es decir, $h_T = z$. El cual ser la representacin vectorial de toda la oracin.

Ahora que tenemos nuestro vector de contexto $z$, podemos empezar a decodear para obtener la oracin target, "good morning". De nuevo, agregamos los tokens de inicio y fin de la secuencia de nuestra oracin target. En cada paso, el input al decoder RNN (cuadros azules de la imagen) es la versin embedding $d$ de la palabra actual $d(y_t)$ as como tambin el hidden state del paso previo $s_{t-1}$m donde el hidden state del decoder incial $s_0$ es el vector de contexto $s_0 = z = h_T$, es decir, el hidden state decoder es el ltimo hidden state encoder. Por ende, simlar al encoder, podemos representarlo como:

$$s_t = \text{DecoderRNN}(d(y_t), s_{t-1})$$

A pesar que el input embeeding layer $e$ y el target embedding layer $d$ estn representados como cuadros amarillos en la imagen, como dijimos en clase, estas son dos embedding layers diferentes con sus propios parametros.

En el decoder, necestamos ir del hidden state a la palabra actual, por ello en cada paso usamos $s_t$ para predecir (a traves de pasarlo en una layer lineal, mostrada como cuadros morados) lo que se cree que es la siguiente palabra en la secuencia $\hat{y}_t$

$$\hat{y}_t = f(s_t)$$

Las palabras en el decoder son siempre generadas una despus de la otra, con una por paso. Siempre usamos <sos> para el primer input del decodr $y_1$ y algumas veces usamos la palabra predicha por nuestro decoder, $\hat{y}_{t-1}$. Que, como mencionamos en clase, se le llama teacher forcing.

Cuando estamos entrenando o probando nuestro modelo, siempre sabemso cuantas palabras hay en nuestra secuencia target, entonces nos detenemos de generar palabras una vez alcanzamos esa cantidad. Durante las fases de inferencia (uso del modelo en la "vida real") seguimos generando palabras hasta que el modelo genere un token <eos> o despus de una cierta cantidad de palabras dada. (Esto tambien lo mencionamoos en clase, es solo para refrescar los conceptos)

Una vez tengamos nuestra secuencia target predicha $\hat{Y} = \{ \hat{y}_1, \hat{y}_2, ..., \hat{y}_T \}$, la comparamos contra nuestra secuencia target real. $Y = \{ y_1, y_2, ..., y_T \}$, para calcular la perdida. Usamos esta prdida para actualizar los parmetros del modelo, como bien hemos hecho en otras ocasiones.

Construyendo el Modelo Seq2Seq

Vamos a definir nuestro modelo en tres partes, el encoder, el decoder y el modelo Seq2Seq. Este ultimo encapsular el proceso y transferencia entre los primeros dos.

Encoder

Primero, el encoder, es un LSTM de 2 capas. El paper que estamos implementando usa un LSTM de 4 capas, pero en favor del tiempo de entrenamiento lo reducimos a 2 capas. El concepto de RNN multicapa es fcil de expandir de 2 a 4 capas.

Para un RNN multicapa, la oracin de entrada, $X$, despus de ser embeddida va a la primera capa (inferior) del RNN y los estados ocultos, $H=\{h_1, h_2, ..., h_T\}$ , la salida de esta capa se utiliza como entrada a la RNN en la capa superior. As, representando cada capa con un superndice, los hidden states en la primera capa vienen dados por:

$$h_t^1 = \text{EncoderRNN}^1(e(x_t), h_{t-1}^1)$$

Las hidden states en la segunda layer son dadas por:

$$h_t^2 = \text{EncoderRNN}^2(h_t^1, h_{t-1}^2)$$

El uso de un RNN multicapa tambin significa que tambin necesitaremos un hidden state inicial como entrada por capa, $h_0^l$, y tambin generaremos un vector de contexto por capa, $z^l$.

Si desean repasar un poco sobre LSTM pueden consultar este [enlance] (https://colah.github.io/posts/2015-08-Understanding-LSTMs/) Para este laboratorio, es suficiente que recuerden que lo que necesitamos saber es los LSTM, en lugar de simplemente tomar un estado oculto y devolver un nuevo estado oculto por paso de tiempo, tambin toman y devuelven un estado de celda, $c_t$, por paso de tiempo.

$$\begin{align*} h_t &= \text{RNN}(e(x_t), h_{t-1})\\ (h_t, c_t) &= \text{LSTM}(e(x_t), h_{t-1}, c_{t-1}) \end{align*}$$

Podemos pensar en $c_t$ como otro tipo de hidden state. Similar a $h_0^l$, $c_0^l$ se inicializar en un tensor de ceros. Adems, nuestro vector de contexto ahora ser tanto el hidden state final como el estado de celda final, es decir, $z^l = (h_T^l, c_T^l)$.

Al extender nuestras ecuaciones multicapa a LSTM, obtenemos:

$$\begin{align*} (h_t^1, c_t^1) &= \text{EncoderLSTM}^1(e(x_t), (h_{t-1}^1, c_{t-1}^1))\\ (h_t^2, c_t^2) &= \text{EncoderLSTM}^2(h_t^1, (h_{t-1}^2, c_{t-1}^2)) \end{align*}$$

Observen cmo solo nuestro hidden state de la primera capa se pasa como entrada a la segunda capa, y no el estado de la celda.

Decoder

Ahora pasaremos a construir el decoder, el cual tambin ser una 2-layer (4 en el paper) LSTM.

La clase Decoder hace un solo paso de decodificacin, es decir, genera un solo token por paso. La primera capa recibir un hidden state y de celda del paso de tiempo anterior, $(s_{t-1}^1, c_{t-1}^1)$, y lo alimenta a travs del LSTM con el token incrustado actual, $y_t$, para producir un nuevo hidden state y de celda, $(s_t ^1, c_t^1)$. Las capas subsiguientes usarn el estado oculto de la capa inferior, $s_t^{l-1}$, y los estados ocultos y de celda anteriores de su capa, $(s_{t-1}^l, c_{t-1) }^l)$. Esto proporciona ecuaciones muy similares a las del codificador.

$$\begin{align*} (s_t^1, c_t^1) = \text{DecoderLSTM}^1(d(y_t), (s_{t-1}^1, c_{t-1}^1))\\ (s_t^2, c_t^2) = \text{DecoderLSTM}^2(s_t^1, (s_{t-1}^2, c_{t-1}^2)) \end{align*}$$

Recuerde que los estados iniciales ocultos y de celda de nuestro decoder son nuestros vectores de contexto, que son los estados finales ocultos y de celda de nuestro decoder de la misma capa, es decir, $(s_0^l,c_0^l)=z^l=(h_T^l,c_T^l)$.

Luego pasamos el hidden state desde la capa superior del RNN, $s_t^L$, a travs de una capa lineal, $f$, para hacer una prediccin de cul ser el siguiente token en la secuencia de destino (salida). debera ser, $\hat{y}_{t+1}$.

$$\sombrero{y}_{t+1} = f(s_t^L)$$

Los argumentos y la inicializacin son similares a la clase Encoder, excepto que ahora tenemos un output_dim que es el tamao del vocabulario para la salida/objetivo. Tambin est la adicin de la capa 'Lineal', utilizada para hacer las predicciones desde el hidden state de la capa superior.

Dentro del mtodo forward, aceptamos un batch de tokens de entrada, hidden state anteriores y estados de celda anteriores. Como solo estamos decodificando un token a la vez, los tokens de entrada siempre tendrn una longitud de secuencia de 1. "Aflojamos" los tokens de entrada para agregar una dimensin de longitud de oracin de 1. Luego, de forma similar al encoder, pasamos a travs de una capa de embedding y aplicamos dropout. Este batch de tokens embeddidos luego se pasa al RNN con los estados ocultos y de celda anteriores. Esto produce una "salida" (hidden state de la capa superior de la RNN), un nuevo "hidden state" (uno para cada capa, apilados uno encima del otro) y una nueva "celda". estado (tambin uno por capa, apilados uno encima del otro). Luego pasamos la salida (despus de deshacernos de la dimensin de longitud de la oracin) a travs de la capa lineal para recibir nuestra prediccin. Luego devolvemos la prediccin, el nuevo hidden state y el nuevo estado celular.

Para esta implementacin, debemos asegurarnos de que el nmero de capas y las dimensiones ocultas (y de celda) sean iguales en el 'Encoder' y 'Decoder'. Este no es siempre el caso, no necesariamente necesitamos la misma cantidad de capas o los mismos tamaos de dimensiones ocultas en un modelo de sequence to sequence. Sin embargo, si hiciramos algo como tener un nmero diferente de capas, tendramos que tomar decisiones sobre cmo manejar esto. Por ejemplo, si nuestro encoder tiene 2 capas y nuestro decoder solo tiene 1, cmo se maneja esto? Promediamos los dos vectores de contexto generados por el decoder? Pasamos ambos por una capa lineal? Solo usamos el vector de contexto de la capa ms alta? Etc.

Nuestro mtodo "forward" toma la oracin fuente, la oracin objetivo y un ratio de teacher-forcing. El ratio de teacher-forcing se usa cuando entrenamos nuestro modelo. Al decodificar, en cada paso, predeciremos cul ser el prximo token en la secuencia de destino de los tokens anteriores decodificados, $\hat{y}_{t+1}=f(s_t^L)$. Con una probabilidad igual a la tasa de teacher forcing (teacher_forcing_ratio), utilizaremos el siguiente token real de la secuencia como entrada al decoder durante el siguiente paso. Sin embargo, con probabilidad 1 - Teacher_forcing_ratio, usaremos el token que el modelo predijo como la prxima entrada al modelo, incluso si no coincide con el siguiente token real en la secuencia.

Lo primero que hacemos en el mtodo forward es crear un tensor outputs que almacenar todas nuestras predicciones, $\hat{Y}$.

Luego alimentamos la oracin de entrada/fuente, src, en el encoder y recibimos los estados ocultos y de celda finales.

La primera entrada al decoder es el token de inicio de secuencia (<sos>). Como nuestro tensor trg ya tiene el token <sos> agregado (desde cuando definimos el init_token en nuestro campo TRG) obtenemos nuestro $y_1$ cortndolo. Sabemos qu tan largas deben ser nuestras oraciones de destino (max_len), por lo que las repetimos muchas veces. El ltimo token ingresado en el decoder es el antes del token <eos> - el <eos> el token nunca se ingresa en el decoder.

Durante cada iteracin del ciclo, nosotros:

  • pasar la entrada, los estados de celda anteriores ocultos y anteriores ($y_t, s_{t-1}, c_{t-1}$) al decoder
  • recibir una prediccin, el siguiente estado oculto y el siguiente estado de celda ($\hat{y}_{t+1}, s_{t}, c_{t}$) del decoder
  • colocar nuestra prediccin, $\hat{y}_{t+1}$/output en nuestro tensor de predicciones, $\hat{Y}$/outputs
  • decidir si vamos a "fuerza de maestros" o no
    • si lo hacemos, la siguiente 'entrada' es el siguiente token de verdad fundamental en la secuencia, $y_{t+1}$/trg[t]
    • si no lo hacemos, la siguiente entrada es el siguiente token predicho en la secuencia, $\hat{y}_{t+1}$/top1, que obtenemos al hacer un argmax sobre el tensor de salida

Una vez que hemos hecho todas nuestras predicciones, devolvemos nuestro tensor lleno de predicciones, $\hat{Y}$/outputs.

Nota: nuestro ccilo decodificador comienza en 1, no en 0. Esto significa que el elemento 0 de nuestro tensor de salidas sigue siendo todo ceros. As que nuestras trg y outputs se parecen a:

$$\begin{alinear*} \text{trg} = [<sos>, &y_1, y_2, y_3, <eos>]\\ \text{resultados} = [0, &\hat{y}_1, \hat{y}_2, \hat{y}_3, <eos>] \end{align*}$$

Posteriormente cuando calculamos la prdida, cortamos el primer elemento de cada tensor para obtener:

$$\begin{alinear*} \text{trg} = [&y_1, y_2, y_3, <eos>]\\ \text{salidas} = [&\hat{y}_1, \hat{y}_2, \hat{y}_3, <eos>] \end{align*}$$

    def __init__(self, encoder, decoder, device):
        super().__init__()

        self.decoder = decoder
        self.device = device

            "Hidden dimensions of encoder and decoder must be equal!"
        assert encoder.n_layers == decoder.n_layers, \
            "Encoder and decoder must have equal number of layers!"
        
    def forward(self, src, trg, teacher_forcing_ratio = 0.5):

def epoch_time(start_time, end_time):
    elapsed_time = end_time - start_time
    elapsed_mins = int(elapsed_time / 60)
    elapsed_secs = int(elapsed_time - (elapsed_mins * 60))
    return elapsed_mins, elapsed_secs
